Sección 01

El DIV, la Atomización y el Float

El <div> es la unidad neutral de maquetación: no tiene semántica propia ni estilos por defecto. Es un contenedor puro que adopta exactamente las propiedades que le asignamos. Combinado con clases atómicas y float, permite construir cualquier layout complejo con CSS puro.

El modelo objeto: etiqueta = instancia

En el DOM, cada etiqueta HTML es una instancia de una clase de objeto. El <div> instancia HTMLDivElement. CSS asigna propiedades a esos objetos mediante selectores. La etiqueta declara la existencia del objeto; CSS define su apariencia y comportamiento visual.

🏗️
HTML
<div class="card">
Instancia el objeto en el DOM
🎨
CSS
.card { width: 300px }
Asigna propiedades al objeto
📐
Selector
.card, #hero, div
Apunta a uno o varios objetos
¿Por qué el <div>? A diferencia de <p>, <h1> o <table>, el <div> no impone nada: sin márgenes, sin padding, sin semántica. Es el lienzo en blanco ideal para maquetación. Su equivalente inline es <span>.

Regla de oro: usá <div> cuando el contenedor es puramente estructural. Usá etiquetas semánticas (<header>, <section>, <article>) cuando el bloque tiene significado de contenido. Para maquetación, el <div> es siempre tu primera opción.

Atomización de clases CSS

La atomización consiste en dividir los estilos en clases pequeñas, de responsabilidad única, que se combinan en el HTML. Esto produce CSS altamente reutilizable y un HTML que describe visualmente el elemento.

Principio de responsabilidad única: cada clase hace una sola cosa.
En lugar de .tarjeta-grande-azul-centrada (monolítica), creás:
.card → estructura base  |  .card--lg → variante de tamaño  |  .bg-violet → color  |  .text-center → alineación

Ventaja: podés combinar libremente. <div class="card card--lg bg-violet"> describe exactamente qué es ese elemento.
CSSatomic-classes.css
/* ── BASE ─────────────────────────────────────────── */
.card       { border-radius: 8px; overflow: hidden; }
.card--sm   { width: 160px; height: 100px; }
.card--md   { width: 240px; height: 140px; }
.card--lg   { width: 320px; height: 200px; }

/* ── COLORES (propiedad única) ─────────────────────── */

.bg-red     { background: #ff6584; }
.bg-green   { background: #22c55e; }
.bg-yellow  { background: #f59e0b; }
.bg-cyan    { background: #06b6d4; }
.bg-dark    { background: #1c2235; }

/* ── ESPACIADO ─────────────────────────────────────── */

.p-lg  { padding: 24px; }

.m-sm  { margin: 8px;  }
.m-md  { margin: 16px; }
.m-auto{ margin-left: auto; margin-right: auto; }

/* ── DISPLAY ───────────────────────────────────────── */

/* ── LAYOUT UTILITIES ──────────────────────────────── */
.w-full  { width: 100%; }
.w-half  { width: 50%;  }
.w-third { width: 33.33%; }
.clearfix::after { content:''; display:block; clear:both; }

⚠️ Anti-patrón: no crear clases como .caja-violeta-grande-con-margen. Si cada variación requiere una clase nueva, el CSS crece sin control. La atomización rompe ese problema.
✔ Buena práctica: empezá siempre por la clase base con la estructura mínima (border-radius, overflow:hidden). Las variantes de color, tamaño y espaciado son clases separadas que se agregan encima.

Box Model — la caja de todo elemento

Cada elemento HTML genera una caja rectangular compuesta por cuatro capas concéntricas. Dominar este modelo es la condición para predecir exactamente cuánto espacio ocupa cada elemento.

Las 4 capas del Box Model

PropiedadValores claveDescripción

CSSreset-global.css
/* Reset recomendado — aplicar siempre al inicio */
*, *::before, *::after {
  box-sizing: border-box;   /* border-box en TODOS los elementos */
  margin: 0;
  padding: 0;
}

/* Con border-box: width: 200px = content + padding + border = 200px TOTAL */
/* Sin border-box (content-box): el elemento crece con padding y border */

.ejemplo {
  width: 200px;    /* ancho total garantizado con border-box */
  padding: 20px;   /* no suma al ancho */
  border: 2px solid #6c63ff; /* no suma al ancho */
}
Colapso de márgenes verticales: cuando dos elementos bloque se tocan verticalmente, sus márgenes no suman — el más grande "gana". margin-bottom: 20px + margin-top: 30px = 30px de separación, no 50px. Esto no ocurre con márgenes horizontales ni en contextos flex o grid.

margin: 0 auto centra un bloque con width definido distribuyendo el espacio sobrante en partes iguales a izquierda y derecha.

Float & Clear — el sistema de layout clásico

Antes de Flexbox y Grid, float era el único mecanismo para construir layouts multi-columna. Entenderlo es obligatorio para mantener código legado y para comprender por qué CSS evolucionó.

¿Qué hace float?
Saca al elemento del flujo normal y lo empuja al borde indicado (left o right). Los elementos que lo siguen en el HTML lo "rodean" visualmente — exactamente el comportamiento de texto alrededor de una imagen.

Problema: el padre colapsa. Si todos los hijos de un contenedor están flotados, el padre no "ve" su altura y colapsa a 0. Necesitás un clearfix.
PropiedadValoresUso
floatleft right noneFlota el elemento al borde indicado.
clearleft right both noneObliga al elemento a bajar hasta que no haya floats del lado indicado.
overflow: hidden—Clearfix simple: fuerza al padre a "envolver" los hijos flotados.
display: flow-root—Clearfix moderno: crea Block Formatting Context sin efectos secundarios.
::after clearfix—Inserta un bloque invisible con clear:both al final del padre.
CSSclearfix-patterns.css
/* ── Método 1: overflow hidden (simple, efectos secundarios) ── */
.wrapper { overflow: hidden; }

/* ── Método 2: display flow-root (RECOMENDADO, sin efectos) ── */
.wrapper { display: flow-root; }

/* ── Método 3: pseudo-elemento ::after (compatible IE) ──────── */
.clearfix::after {
  content: '';
  display: block;
  clear: both;
}

/* ── Layout 2 columnas con float ────────────────────────────── */
.col-left  { float: left;  width: 30%; }
.col-right { float: left;  width: 70%; }  /* también left, no right */

/* ── Columnas con gap: usá margin en lugar de gap ───────────── */
.col-left  { float: left; width: 28%; margin-right: 2%; }
.col-right { float: left; width: 70%; }

/* ── Forzar elemento a estar DEBAJO de todos los floats ──────── */
.footer { clear: both; }
Float: colapso del padre vs clearfix aplicado
Sin clearfix: padre colapsa a 0px alto
Con clearfix: padre envuelve los hijos

Position — salir del flujo normal

La propiedad position controla si un elemento participa en el flujo del documento o se posiciona de forma especial. Las propiedades top, right, bottom, left y z-index solo actúan cuando position ≠ static.

ValorReferencia¿En el flujo?Uso típico
static—✔Default. top/left ignorados.
relativeSu posición original✔ (reserva espacio)Pequeños ajustes. Ancla para hijos absolute.
absoluteAncestro posicionado más cercano✗Badges, tooltips, overlays.
fixedViewport✗Headers fijos, botones flotantes.
stickyFlujo normal + viewport al llegar al umbral✔Headers que se fijan al hacer scroll.
CSSposition-patterns.css
/* ── Patrón ancla: padre relative, hijo absolute ───── */
.tarjeta    { position: relative; }   /* ancla */
.badge      { position: absolute; top: -6px; right: -6px; }

/* ── z-index: solo funciona con position ≠ static ──── */
.modal      { position: fixed; z-index: 1000; }
.overlay    { position: fixed; z-index: 999; }
.dropdown   { position: absolute; z-index: 100; }

/* ── Header sticky ──────────────────────────────────── */
.header {
  position: sticky;
  top: 0;           /* se fija cuando llega al top del viewport */
  z-index: 50;
}

/* ── Overlay full-screen ────────────────────────────── */
.overlay {
  position: fixed;
  top: 0; left: 0;
  width: 100%; height: 100%;
  background: rgba(0,0,0,.6);
}

Sección 02

Flexbox y Responsividad Total

Flexbox resuelve distribución y alineación en una dimensión con un control que float jamás ofreció. Combinado con Media Queries y Container Queries, permite construir layouts que se adaptan perfectamente a cualquier tamaño de pantalla o contenedor.

El modelo Flexbox — contenedor y elementos

Al declarar display: flex en un elemento, este se convierte en el flex container. Sus hijos directos se convierten automáticamente en flex items. El contenedor controla cómo se distribuyen sus hijos; los hijos controlan su propio comportamiento dentro de ese contexto.

Media Queries — control total por viewport

Una Media Query es una condición que aplica un bloque de CSS solo cuando se cumple. El enfoque mobile-first (con min-width) es el estándar moderno: partís del diseño más pequeño y lo expandís.

Mobile-first (recomendado): escribís el CSS base para mobile. Las media queries con min-width agregan estilos para pantallas más grandes.

Desktop-first: escribís el CSS base para desktop. Las media queries con max-width sobreescriben para pantallas pequeñas. Más difícil de mantener.
📱 mobile: base (0px+)
📲 tablet: min-width 768px
🖥 desktop: min-width 1024px
🖥 wide: min-width 1280px
CSSresponsive-mobile-first.css
/* ══ BASE — mobile (se aplica siempre) ══════════════════════ */
.grid {
  display: flex;
  flex-direction: column;   /* apilado en mobile */
  gap: 12px;
}

.card {
  flex: 1 1 auto;
  min-height: 120px;
  border-radius: 8px;
}

.sidebar { display: none; }  /* oculto en mobile */

/* ══ TABLET — 768px y más ════════════════════════════════════ */
@media (min-width: 768px) {
  .grid {
    flex-direction: row;    /* fila en tablet */
    flex-wrap: wrap;
  }
  .card { flex: 1 1 45%; }  /* 2 columnas */
}

/* ══ DESKTOP — 1024px y más ══════════════════════════════════ */
@media (min-width: 1024px) {
  .grid { flex-wrap: nowrap; }
  .card { flex: 1 1 0; }    /* 3+ columnas iguales */
  .sidebar { display: flex; flex: 0 0 240px; }  /* sidebar visible */
}

/* ══ Orientación del dispositivo ════════════════════════════ */
@media (orientation: landscape) and (max-width: 768px) {
  .hero { min-height: 50vh; }   /* más bajo en landscape mobile */
}

/* ══ Rango de breakpoints (min + max) ═══════════════════════ */
@media (min-width: 480px) and (max-width: 767px) {
  .card { flex: 1 1 48%; }  /* solo en esta franja */
}

Container Queries — responsividad por contenedor

Una limitación de las Media Queries es que reaccionan al viewport. Un componente reutilizable en un sidebar y en un contenedor ancho necesitaría media queries distintas. Container Queries resuelven esto: el componente reacciona al tamaño de su propio contenedor padre.

Pasos:
1. Declarar el contenedor padre: container-type: inline-size; (y opcionalmente container-name: nombre).
2. Escribir la query: @container (min-width: 400px) { … }

Cuándo usar Container Queries vs Media Queries:
— Media Query: cuando el cambio depende del tamaño total de la pantalla (columns del layout global, visibilidad del sidebar).
— Container Query: cuando el cambio depende del espacio disponible para ese componente específico (tarjeta, card, widget).
CSScontainer-queries.css
/* ── 1. Declarar el contenedor ──────────────────────────────── */
.card-wrapper {
  container-type: inline-size;
  container-name: card;       /* nombre opcional */
}

/* ── 2. Estilos base de la card (mobile-first) ──────────────── */
.card-inner {
  display: flex;
  flex-direction: column;     /* apilado cuando es angosto */
  background: #1c2235;
  border-radius: 8px;
  overflow: hidden;
}

.card-thumb {
  height: 120px;
  background: #6c63ff;
}

.card-body {
  padding: 16px;
  background: #232a3e;
  flex: 1;
}

/* ── 3. Query: cuando el CONTENEDOR tiene 400px+ ────────────── */
@container card (min-width: 400px) {
  .card-inner {
    flex-direction: row;      /* lado a lado cuando hay espacio */
  }
  .card-thumb {
    width: 160px;
    height: auto;
    flex-shrink: 0;
  }
}

/* En sidebar: card apilada verticalmente                         */
/* En main:    card horizontal — sin escribir ningún media query  */

Buenas prácticas de responsividad

✔ Siempre mobile-first: escribí el CSS base para el viewport más pequeño y usá min-width para escalar. Es más fácil agregar complejidad que quitarla.
✔ Usá flex: 1 1 Xpx en lugar de anchos fijos: flex: 1 1 200px con flex-wrap: wrap crea grillas responsivas sin media queries.
✔ Unidades relativas para espaciado: gap: 1rem, padding: 5%, max-width: 1200px; margin: 0 auto para contenedores.
✔ Container Queries para componentes: no escribas media queries que apunten al viewport para adaptar un componente interno. Usá container-type.
⚠️ No mezcles unidades arbitrariamente: definí un set de tokens de espaciado en :root y usalos consistentemente con variables CSS.
⚠️ Evitá breakpoints "raros": @media (max-width: 753px) probablemente está tapando un problema de layout que se debería resolver con flex-wrap o minmax.
CSSbest-practices.css
/* ── Tokens de diseño en :root ─────────────────────────────── */
:root {
  --space-xs:  4px;
  --space-sm:  8px;
  --space-md:  16px;
  --space-lg:  24px;
  --space-xl:  40px;
  --space-2xl: 64px;

  --radius-sm: 4px;
  --radius-md: 8px;
  --radius-lg: 16px;

  --max-width: 1200px;
}

/* ── Contenedor centrado responsivo ────────────────────────── */
.container {
  width: 100%;
  max-width: var(--max-width);
  margin: 0 auto;
  padding: 0 var(--space-md);
